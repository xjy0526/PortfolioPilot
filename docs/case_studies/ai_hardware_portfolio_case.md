# AI 硬件主题组合风险诊断与调仓实验

> 本案例仅用于研究、系统演示和工程验证，不构成投资建议、交易建议或任何收益承诺。

## 1. 案例目标

本案例演示 PortfolioPilot 如何对一个 AI 硬件主题组合完成端到端分析：

- 组合风险摘要 `risk summary`
- 本地 RAG 证据检索
- LLM 结构化 JSON 分析
- 风险调整调仓建议
- 多策略回测对比

案例主题是 **AI 硬件主题组合风险诊断与调仓实验**。组合覆盖 PCB、CPO、芯片、ETF 和现金，但这些资产的共同驱动都与 AI 服务器、半导体资本开支、数据中心建设和供应链周期有关，因此系统应重点识别主题集中风险。

## 4. Risk Summary 结果

系统生成的组合风险摘要核心结果如下：

| 指标 | 结果 |
| --- | ---: |
| `risk_score` | `7.4 / 10` |
| `risk_level` | `high` |
| 期间收益 | `7.2%` |
| 年化波动率 | `74.7%` |
| 最大回撤 | `-9.7%` |
| Sharpe Ratio | `8.00` |

注意：本案例行情是短样本演示数据，年化收益和 Sharpe 会被短期窗口放大，不应被解释为真实投资结论。这里更关注系统链路是否能识别风险。

### 4.1 单资产集中度

系统识别到最大单资产为：

| 资产 | 权重 |
| --- | ---: |
| `002916.SZ` | `28.2%` |
| `300476.SZ` | `22.9%` |
| `300308.SZ` | `12.3%` |
| `NVDA` | `8.7%` |
| `688498.SS` | `8.5%` |

其中 `002916.SZ` 超过 25% 单资产研究阈值，因此触发：

```text
single_asset:002916.SZ:28.2%
```

这表示系统把它作为组合层面的单资产集中风险，而不是简单地看作普通持仓。

### 4.2 行业集中度

按行业/主题链路聚合后，系统得到：

| 行业/子主题 | 权重 | 持仓数 |
| --- | ---: | ---: |
| AI Hardware - PCB | `51.1%` | 2 |
| AI Hardware - CPO | `20.8%` | 2 |
| AI Hardware - Chips | `14.2%` | 3 |
| AI Hardware - ETF | `8.4%` | 2 |
| Cash | `5.5%` | 1 |

系统触发行业集中风险：

```text
sector:AI Hardware - PCB:51.1%
```

这说明虽然组合表面上包含 PCB、CPO、芯片和 ETF，但 PCB 子主题本身占比已经超过一半，主题内部并不均衡。

### 4.3 资产类型暴露

资产类型暴露如下：

| 资产类型 | 权重 |
| --- | ---: |
| China A-Share | `71.9%` |
| US Equity | `14.2%` |
| ETF | `8.4%` |
| Cash | `5.5%` |

系统可以看到组合对 A 股 AI 硬件链路高度暴露，同时 ETF 又可能与芯片个股形成重复因子暴露。

### 4.4 波动和回撤风险

历史价格样本显示组合年化波动率为 `74.7%`、最大回撤为 `-9.7%`。在短样本中，这主要来自 PCB、CPO 和芯片标的同步波动。

系统并不把“包含多个资产”直接等同于“充分分散”，而是结合价格序列、行业分组和资产权重判断主题相关性。

这些 evidence 共同提供了以下上下文：

- AI 服务器链路可能存在同一主题 beta；
- PCB 与 CPO 子行业都可能受订单验证、价格竞争和库存周期影响；

RAG 证据不是直接给出买卖判断，而是帮助 LLM 分析模块解释为什么系统应关注集中度、波动和回撤。

## 7. 调仓建议解释

LLM 风险调整权重模块输出的是研究型目标权重，不执行真实交易。

重点结果：

| 资产 | 当前权重 | 目标权重 | 变化 | 解释 |
| --- | ---: | ---: | ---: | --- |
| `002916.SZ` | `28.2%` | `25.0%` | `-3.2%` | 单资产超过 25%，且组合风险评分较高 |
| `300476.SZ` | `22.9%` | `24.3%` | `+1.4%` | 在单资产上限约束下重新归一化 |
| `300308.SZ` | `12.3%` | `13.4%` | `+1.2%` | 同属高风险主题，保留观察而非交易指令 |
| `NVDA` | `8.7%` | `9.8%` | `+1.1%` | 主题高波动，按风险模型重新分配 |
| `688498.SS` | `8.5%` | `9.6%` | `+1.1%` | 高风险资产，需结合 evidence 复核 |

系统同时给出行业集中提示：

```text
Sector concentration warning: AI Hardware - PCB is 51.1%, above the 40% research threshold.
```

解释重点：

1. 系统并不是简单地“卖出高风险资产”。
2. 它先对单资产权重做上限约束，再对剩余资产做归一化。
3. 因为整个组合几乎都是 AI 硬件主题，部分资产目标权重上升并不代表投资建议，只代表在模型约束下的权重再分配。
4. 真实使用时应把这些结果作为研究线索，而不是交易指令。

## 9. 系统识别逻辑总结

### 行业集中度

系统按 `sector` 聚合当前市值，发现 `AI Hardware - PCB` 达到 `51.1%`。这触发行业集中提示，说明组合虽然有多个 AI 硬件环节，但 PCB 子主题过度集中。

### 单资产集中度

系统计算资产权重，发现 `002916.SZ` 达到 `28.2%`。超过 25% 阈值后，调仓模块将其目标权重压到 `25.0%`。

### 波动风险

系统从价格序列计算收益率、年化波动和最大回撤。PCB、CPO、芯片资产在短样本中同步波动，组合年化波动率达到 `74.7%`，因此风险等级为 `high`。

### 调仓建议

调仓模块将风险摘要和结构化分析结果转化为目标权重。它输出的是：

- `target_weight`
- `weight_change`
- `reason`

这些字段用于前端展示和研究复核，不用于真实交易。

## 10. 结论

本案例展示了 PortfolioPilot 对 AI 硬件主题组合的完整研究链路：

1. 从本地 CSV 读取主题组合；
2. 用历史价格生成组合风险摘要；
3. 用 RAG 检索本地研究证据；
4. 用结构化 LLM 分析产出 JSON 风险解释；
5. 用调仓算法生成可解释目标权重；
6. 用真实 CSV 行情完成策略回测对比。

整个案例只用于系统演示和研究流程验证，不构成任何投资建议。
